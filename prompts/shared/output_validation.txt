## JSON 출력 검증 규칙

### 출력 전 필수 체크리스트
응답을 생성하기 전에 다음 사항을 반드시 확인하세요:

1. **JSON 형식 유효성**
   - 유효한 JSON 구조인지 확인 (중괄호, 대괄호 매칭)
   - 문자열은 큰따옴표(") 사용
   - 마지막 항목 뒤에 쉼표(,) 없음
   - 특수문자 이스케이프 처리 (\n, \", \\)

2. **필수 필드 존재**
   - 명세서에 정의된 모든 필수 필드 포함
   - 필드명 정확히 일치 (대소문자, 철자)
   - 빈 값이 허용되지 않는 필드는 적절한 기본값 제공

3. **데이터 타입 일치**
   - 문자열은 "", 숫자는 따옴표 없음
   - 배열은 [], 객체는 {}
   - Boolean은 true/false (소문자, 따옴표 없음)

4. **값 범위 검증**
   - 숫자 범위가 지정된 경우 준수
   - 열거형(enum) 값은 허용된 값만 사용
   - 배열 길이가 지정된 경우 준수

### 일반적인 오류 패턴 (피할 것)

❌ **잘못된 예시:**
```json
{
  "field1": "value1",   // 주석 포함 불가
  "field2": 'value2',   // 작은따옴표 사용 불가
  "field3": undefined,  // undefined 사용 불가
  "field4": NaN,        // NaN 사용 불가
}                       // 마지막 쉼표 불가
```

✅ **올바른 예시:**
```json
{
  "field1": "value1",
  "field2": "value2",
  "field3": null,
  "field4": 0
}
```

### 응답 포맷팅
- **순수 JSON만 반환**: Markdown 코드 블록(```)이나 설명 텍스트 없이 JSON만 출력
- **들여쓰기**: 가독성을 위해 2칸 또는 4칸 들여쓰기 사용
- **필드 순서**: 명세서의 필드 순서 유지 (필수는 아니지만 권장)

### 에러 처리
요청을 처리할 수 없는 경우에도 유효한 JSON 응답을 반환하세요:

```json
{
  "success": false,
  "error": "오류 메시지",
  "error_code": "ERROR_CODE",
  "details": "추가 설명"
}
```
